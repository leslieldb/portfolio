Worksample - PikUp

View code on GitHub

Task — Create an Android application to help people connect and build their community. In this application, you can game-ify picking up trash to minimize litter, you can also pick up and donate nearby tools to share, and split a shopping bill with someone in exchange for a share of the groceries, to help minimize food insecurity and food waste.

Steps:

  • Create a SQL database for the application to store user data, location data, co-shop data, tool data, and trash pickup data.
  • Connect the Android Studio application to the database.
  • Create a Login page and account system.
  • Create a Dashboard page, displaying the user's points, and a leaderboard of the top 10 users' scores.
  • Create a Profile page
  • Import an external map service, to display a map of the user's location inside the application
  • Create the Trash Cleanup page and functionality
  • Create the Tool Share page and functionality
  • Create the Co-Shop page and functionality, based on the winning 2023 Challenge Nova Scotia proposal, which you can find here
  • Create a product details page, which will display all information of a selected product
  • Create payment functionality

My Duties:

  • Designing and implementing the Dashboard page and its SQL queries
  • Designing and implementing the Product Details page and its SQL queries
  • Optimizing the tables and fields in the database
  • Troubleshooting and testing

Skill area — User Experience (UX)

Work Example

Key Learnings:

  • Technical skills used:
    • Kotlin, SQL, XML, Figma, Systems Analysis
  • Progression of learning:
    • The most I've ever done coding an Android app before was plotting bus locations on a map. This was a big step up, and opened my eyes to the fact that there is so much more to all of these languages that we don't learn in college, and that we don't even fathom is possible. Neverless, my team and I were able to put together a mostly functional prototype of the PikUp application, and we're all very proud of ourselves.
  • How would I make it better?
    • I didn't have much time to work on this application at all over the three months, and I didn't do as much as I hoped I would be able to. I had gotten a lot better at time management before working on this project, and it showed, but I think I have to work on it even more. I would have felt a lot more satisfied in the end.

Business benefits:

  • Because of the encouraged community engagement, brand awareness, and ease-of-use of this application, this shows that I can contribute to a team that develops programs that:
    • Can greatly improve communities
    • Can help enhance the team's reputation
    • Can create many networking opportunities
    • Have very user-friendly interfaces

Connection to Me:

  • It felt really nice that an idea that my peers and I came up with was being implemented into a big group capstone, even with the flaws the idea had. I'm very happy with how the first prototype of Co-Shop turned out, and I would love to take it a bit further.
  • This was my first ever group coding project, and it was a very nice first impression, even with all of the setbacks. I'm looking forward to collaborating with even more wonderful programmers in the future!

Worksample - Countries of the World

View code on GitHub

Task — Develop a GUI program that reads a list of countries and their information from a file, and when clicked, displays its name, flag, population, area in miles and kilometers, and population density in miles and kilometers. Population can be updated by the user and can be saved and overwritten to a file.

Steps:

  • Created a new interface in Python Qtdesigner, using multiple selection lists and buttons.
  • Set up the graphical buttons with normal Python functions, to do things such as update a country’s population density or write the information to a file.
  • Thoroughly tested the program for errors or bugs in the software, to ensure peak performance.

Skill area — Data Visualization

Work Example

Key Learnings:

  • Technical skills used:
    • Python, UI Design, Systems Analysis
  • Progression of learning:
    • With minimal knowledge on UI design and Python UI, I was able to create a program which displayed accurate information about countries read from a file.
  • How would I make it better?
    • Looking back, I realized that some information displayed was inaccurate because of a bug in the program that I didn’t find while testing. To make it better, I would have tested the program a lot more thoroughly so the program would meet all needs. I also would have applied best practices for designing a UI, and would have put every program action in its own function as that is better practice.

Business benefits:

  • Because of the program design, this shows that I can develop programs that:
    • Meet client needs
    • Are more compact and efficient
    • Are easier to understand
    • Are a good demonstration of teamwork

Connection to Me:

  • This was a great demonstration of my learning, and also shows how quickly I was able to pick up on it!
  • I've become a bit of a geography buff, so I was very motivated to work on this as this ties into that. This shows off that as long as I'm interested, I can stay very self-motivated to get things done.
  • This work sample helped me realize that I need to test programs more thoroughly. I have since then taken a course containing program testing, so I've learned even more about the importance of testing.

Worksample - Ecosystem Simulation

View code on GitHub

Task — Develop a console-based program that simulates an ecosystem containing Humans and Hombies in a 20 x 20 grid. Application should run until either species goes extinct. Humans should be able to move and recruit other Humans, while Zombies should be able to move, breed other Zombies, and bite Humans to turn them into Zombies. Zombies should also be able to starve and change back to a Human. Number of Humans and Zombies should be constantly tracked.

Steps:

  • Display a 20 x 20 grid, and fill the grid with an initial amount of entities.
  • Create a turn function for both Humans and Zombies, and run functions for every possibly action inside each turn function.
  • Ensure that the ecosystem functions correctly, and terminates if an extinction happens.
  • Thoroughly test the program for errors or bugs, to ensure peak performance.

Skill area — Object-Oriented Programming

Work Example

Key Learnings:

  • Technical skills used:
    • C++, Systems Analysis
  • Progression of learning:
    • This was a huge step up from anything I had done in C++ before. Even so, I was able to create a program which successfully simulated a working ecosystem in the console.
  • How would I make it better?
    • I had tried to add a few more cosmetic features, but they ended up not working, so I scrapped them. One of these features was turning the entire grid to black text on a bright red background whenever an extinction happened. I would like to come back and tie up these loose ends.

Business benefits:

  • Because of the program architecture, design, and testing, this shows that I can develop programs that:
    • Meet client needs
    • Are quick and efficient
    • Are optimized and tidied well
    • Demonstrate proper polymorphism
    • Are visually appealing to the user

Connection to Me:

  • This was a fun learning experience, and helped me gain lots of confidence in my C++ abilities that I had nowhere near as much of before starting the project.
  • This work sample made me realize that I overthink things a lot. Once I had a tiny nudge in the right direction, I realized that this was a lot simpler than I had initially thought, and I got it done in good time.
